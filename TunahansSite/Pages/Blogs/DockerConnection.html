How to connect your project to Docker?

First one we have to learn to what is Docker?

Before we dive into commands and setups, it’s important to understand what Docker actually is. Docker is a platform that helps developers package applications and their dependencies into a container—a lightweight, portable unit that can run consistently on any system. This means no more “it works on my machine” problems. Whether you’re working on a small project or deploying something at scale, Docker gives you a cleaner, more reliable way to build, ship, and run your software.

So lets start to connect project to Docker?

Now that we know what Docker is, let’s use it. We’ll start by creating a Dockerfile to define how our app should run. Then, we’ll build an image and run it as a container. Step by step, you’ll see how easy it is to make your project portable and consistent across different environments.

PROCESS STEPS:

1) Create a Dockerfile

In your project’s root directory, create a file named Dockerfile (no extension). This will contain the instructions for building your Docker image.

2) Write Dockerfile Instructions

Inside the file, add basic instructions. For example, for a Node.js app:
(Hover the code to see the hidden comment)

1 FROM node:18 //Uses the official Node.js version 18 image as the base environment.

2 WORKDIR /app //Sets the working directory inside the container to /app.

3 COPY . . //Copies everything from your project folder into the container’s /app directory.

4 RUN npm install //Installs all project dependencies inside the container.

5 CMD ["npm", "start"] //Runs npm start when the container starts—this launches your app.

Adjust this according to your tech stack.

3) Build the Docker Image

Open your terminal, navigate to your project folder, and run:

➜   docker build -t my-app . // Builds the Docker Image with the build context.

4) Run the Docker Container

After building, you can run the container with:

➜   docker run -p 3000:3000 my-app // Runs the Docker Container.

(Replace 3000 with your app’s port if it’s different.)

5) Test Your App

Visit http://localhost:3000 in your browser. If everything went well, your app should be running inside Docker!

What is Docker Compose? (Optional Section)

Docker Compose is a tool that lets you manage multi-container Docker applications with ease. Instead of starting each container manually with long docker run commands, you define all your services (like backend, frontend, and database) in a simple docker-compose.yml file. With one command—docker compose up—you can start everything at once. It’s especially useful in development, where multiple services need to run together seamlessly.

PROCESS STEPS:

1) Create a docker-compose.yml file

In your project root, create a file named docker-compose.yml.

2) Define Services

Inside the file, define your app and other services. For example:
(Hover the code to see the hidden comment)

1 version: "3" //Specifies the Docker Compose file format version.

2 services: //Starts the definition of services (containers) to run.

3     app: //Names the service app (you can choose any name).

4        build: . //Tells Docker to build the image from the current directory (uses the Dockerfile).

5        ports:
 - "3000:3000" //Maps port 3000 of your machine to port 3000 in the container (host:container).

3) Build and Start Services

Run this in your terminal:

➜   docker compose up --build // Builds and starts the services defined in docker-compose.yml.

5) Verify It’s Working

Visit http://localhost:3000. Your app should be running!

5) Stop Services

When done, shut everything down with:

➜   docker compose down // Stops and removes the containers defined in docker-compose.yml.
